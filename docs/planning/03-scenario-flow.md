# 3. 시나리오 & 화면 흐름

> ★ 표시는 원본에서 확정된 방향입니다.

## 사용 시나리오 (선택형으로 시작)

| 단계 | 화면/행동 | 설명 | 비고 |
|------|-----------|------|------|
| 1 | 시작 — 선택 입력 | 인원·기간·연령·성향 등을 버튼 선택으로 입력 | ★대화가 아니라 선택으로 시작 |
| 2 | 인증샷 | 친구들과 미션 시작 전 인증샷 촬영·프린트(네컷사진처럼) | 처음 인물이 이후 미션 인증에 사용 |
| 3 | 현장 코스 생성 | 위치·시각·날씨·실시간 영업을 반영해 코스 생성 | ★차별점 |
| 4 | 지도 확인·수정 | 경로를 지도에 표시, 특정 경로 즉시 제거 | |
| 5 | 미션 부여 | 5만원 여행/관광지 스탬프 등 미션 제시 | 선택 사항 |
| 6 | 출력 | QR로 전송 또는 즉석 인쇄 | |
| 7 | 미션 수행 | 관광지에서 인증샷 → 스탬프 → 포인트 | |
| 8 | 종료 | 개인정보 미저장, 세션 폐기 | ★개인정보 안 남김 |

## (원본) 키오스크 형태

| 항목 | 내용 | 비고 |
|------|------|------|
| 설치 장소 | 기차역, 터미널 | 유동 인구 많은 거점 |
| 인증샷/프린트 | 미션 시작 전 인증샷을 네컷사진처럼 출력 | 미션 인증에 활용 |

## 웹앱 관점 참고 노트

- **선택형 입력(★)**: 웹앱에서도 유지. 대화형 대신 버튼/슬라이더 기반 입력으로 빠른 시작.
- **2단계 인증샷**: 키오스크 프린트 대신 웹앱은 카메라(`getUserMedia`)로 촬영 후 이미지 카드 생성.
- **6단계 출력**: 즉석 인쇄 대신 공유 링크/QR/이미지 저장으로 대체.
- **8단계 세션 폐기**: 익명 세션 + TTL 자동 만료로 구현.
